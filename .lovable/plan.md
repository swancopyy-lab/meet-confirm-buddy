
## الخطة

### 1. حسابك كمشرف عام (Super Admin)
- إضافة دور جديد `super_admin` في جدول الأدوار.
- عند تسجيل دخولك بالإيميل `swancopyy@gmail.com` لأول مرة، يُمنح حسابك دور `super_admin` تلقائياً (تحقق بواسطة الإيميل المؤكد فقط لتفادي انتحال الهوية).
- صفحة جديدة `/admin` تظهر فقط للمشرف العام، فيها:
  - قائمة المستخدمين المسجلين مع حالة الموافقة (Approved / Pending / Blocked).
  - أزرار موافقة/حظر/إلغاء.
  - حقل لتحديد الحد الأقصى لعدد الدعوات (الباركودات) لكل مستخدم، مع قيمة افتراضية عامة قابلة للتعديل.

### 2. الموافقة على المستخدمين قبل استخدام المنصة
- عمود `approval_status` في جدول `profiles` بقيم: `pending` / `approved` / `blocked` (افتراضي `pending`).
- بعد التسجيل، المستخدم يرى شاشة "بانتظار موافقة المشرف" ولا يستطيع إنشاء أي مناسبة أو دعوة.
- عند الموافقة يفتح له الوصول تلقائياً بعد إعادة تحميل الصفحة.

### 3. حد أقصى لعدد الدعوات لكل مستخدم
- عمود `invitation_quota` على `profiles` (رقم صحيح، افتراضي مثلاً 100).
- عند توليد دعوات جديدة يتم التحقق من مجموع دعوات المستخدم عبر كل مناسباته:
  - إذا (الحالي + الجديد) > الكوتا → رفض العملية مع رسالة واضحة.
- المشرف العام يعدّل الكوتا لكل مستخدم من صفحة `/admin`.

### 4. إنشاء حساب مساعد من صفحة المناسبة
- في صفحة المناسبة يوجد حالياً إضافة مساعد بالإيميل فقط. سنضيف خيار:
  - **إنشاء حساب جديد للمساعد** بإدخال (إيميل + كلمة مرور) — يتم إنشاء الحساب من طرف الخادم بصلاحيات إدارية ويُضاف مباشرة كمساعد للمناسبة، وحسابه يكون معتمداً تلقائياً (`approved`) وكوتاه = 0 (لأنه لن ينشئ مناسبات خاصة به).
  - المساعد يستقبل الإيميل + كلمة المرور من صاحب المناسبة ويسجل دخوله مباشرة.

### 5. المساعد يرى صفحة إدارة الدعوات فقط، لا صفحة المدعوّين
- حالياً المساعد بإمكانه فتح صفحة المناسبة كاملة. سنقيّد صلاحيات المساعد على النحو:
  - **يستطيع:** رؤية قائمة الدعوات، توليد باركودات جديدة (ضمن كوتا صاحب المناسبة)، مسح الباركود عند الباب، رؤية إحصائيات الحضور.
  - **لا يستطيع:** رؤية بيانات المدعوّين الشخصية (الاسم، الهاتف، الملاحظة، عدد المرافقين المؤكد). ستُخفى هذه الأعمدة من الاستعلامات التي يقوم بها المساعد وتُعرض له فقط: رقم الدعوة/الرمز + حالة الحضور (تم المسح / لم يُمسح).
  - تنبيه: هذا يعني أن ملخص "من قال إنه حاضر" يبقى لصاحب المناسبة فقط، بينما المساعد يرى أرقام إجمالية وحالة المسح.
- تحديث سياسات RLS على `invitations` لإرجاع الأعمدة الحساسة فقط لصاحب المناسبة، وإنشاء دالة/عرض آمن للمساعد يُظهر الأعمدة المسموحة فقط.

### 6. صفحة الدعوة العامة `/i/:code` تبقى كما هي للمدعو
- لا تغيير على تجربة المدعو نفسه: يفتح رابطه، يؤكد الحضور، ويخرج. لا علاقة له بواجهات المساعدين أو المشرف.

## التفاصيل التقنية (للمرجع)

- **قاعدة البيانات (migration واحدة):**
  - `alter type app_role add value 'super_admin';`
  - `alter table profiles add column approval_status text default 'pending' check (in ('pending','approved','blocked'))`, `add column invitation_quota int default 100`.
  - Trigger على `auth.users` (insert/update `email_confirmed_at`) لمنح `super_admin` للإيميل `swancopyy@gmail.com` عند التحقق فقط.
  - Trigger على `profiles` لتعبئة `approval_status='approved'` تلقائياً لحساب المشرف العام وللمساعدين الذين ينشئهم الخادم (نستخدم عمود مؤقت أو نمرر عبر الخدمة).
  - دوال `has_role(super_admin)` تُستخدم في سياسات جديدة على `profiles` تسمح للمشرف العام بقراءة/تحديث الجميع.
  - دالة `get_user_invitation_count(uuid)` تحسب مجموع دعوات المستخدم.
  - تحديث سياسات `invitations` لتقييد أعمدة الحقول الحساسة عبر view آمن باسم `invitations_collaborator_view` مع `security_invoker=on`، ورفض SELECT المباشر على الأعمدة الحساسة للمساعد.

- **Server functions جديدة في `src/lib/`:**
  - `admin.functions.ts`: `listUsers`, `setUserApproval`, `setUserQuota`, `setDefaultQuota` (كلها محمية بـ `super_admin`).
  - في `invitations.functions.ts`: إضافة تحقق الكوتا + حالة الموافقة قبل التوليد، ودالة `createCollaboratorAccount(eventId, email, password)` تستخدم `supabaseAdmin` بعد التحقق من ملكية المناسبة.
  - تعديل دوال قراءة الدعوات لتُرجع للمساعد الأعمدة الآمنة فقط.

- **الواجهة:**
  - صفحة `/admin` جديدة تحت `_authenticated/` مع gate إضافي (`super_admin`).
  - شاشة "بانتظار الموافقة" داخل `_authenticated` عندما `approval_status !== 'approved'` وليس المشرف العام.
  - في صفحة المناسبة: تبويب "المساعدون" يضيف زر "إنشاء حساب مساعد" مع نموذج (إيميل + كلمة مرور).
  - في صفحة المناسبة عند فتحها من قِبل مساعد: إخفاء تبويب/عمود المدعوّين وإظهار قائمة مبسطة (رقم الدعوة + الحالة).
